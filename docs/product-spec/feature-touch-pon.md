# 機能仕様: タッチ反応「ぽん」

再生中のシーン領域を触ると、図形（またはどうぶつ）が短く膨らむ／跳ねる。
赤ちゃん向けの「触ったら動いた」因果反応。セッション進行そのものは変えない。

## 目的

- 保護者が赤ちゃんの手を添えたり、赤ちゃん自身が触れたときに、穏やかな反応を返す
- 点滅や強いフラッシュを使わず、既存モーションの延長として実装する
- 一時停止・終了・時間管理の安全性を壊さない

## 位置付け

| 項目 | 内容 |
|------|------|
| 対象状態 | `playing` のみ |
| 対象領域 | Player の scene（Canvas stage）全体 |
| 対象 pack | 初回は `colors`。`animals` 実装後は両 pack で同じ挙動 |
| 設定項目 | Setup に ON/OFF は設けない（常に有効） |
| セッション FSM | 変更しない。シーン進行・deadline には影響しない |

## 反応仕様

### 入力

| 項目 | 要件 |
|------|------|
| イベント | `pointerdown`（タッチ・マウス・ペンを同一経路で扱う） |
| 座標 | Canvas の CSS ピクセル座標 → pose 正規化座標へ変換 |
| 対象アクター | タップ位置に最も近いアクター。距離が閾値外なら主アクター |
| ヒット判定 | 厳密なピクセルヒットは不要。中心距離ベースでよい |

一時停止中・終了後・Setup では反応しない。コントロールボタン（一時停止・終了）へのポインタは scene に伝播させない。

### 見た目

| 項目 | 要件 |
|------|------|
| 種別 | スケールアップ → 戻るの「ぽん」（任意でごく短い縦方向バウンスを併用可） |
| ピーク倍率 | 1.12〜1.18（これ以上大きくしない） |
| 時間 | ピーク到達 80–120ms、復帰 180–280ms。合計 roughly 300–400ms |
| イージング | 穏やか（overshoot は 1 回まで。連続バウンドや振動はしない） |
| 色変化 | しない（フラッシュ禁止） |
| 不透明度 | 変えない |
| 回転 | 追加の急回転はしない |

反応中も既存の転がり軌道は継続してよい。スケールは軌道 pose に乗算する。

### 連打・安全

| 項目 | 要件 |
|------|------|
| 同一アクターの再発火 | 前回反応の完了まで無視、または完了前の再タップは「残り時間をやり直す」のどちらか一方に固定する（推奨: 完了まで無視） |
| 全体レート制限 | どのアクターでも **250ms に 1 回まで** 新規反応を開始しない |
| 複数指 | 最初の pointer のみ採用。マルチタッチで複数同時反応しない |
| 点滅相当 | 短周期の明滅やサイズ点滅に見える連打反応を禁止する（上記レート制限で担保） |

`prefers-reduced-motion` でも再生中の動きは抑えない（現行 product-spec どおり）。「ぽん」も同様に抑えない。

## 音

- 「ぽん」専用の効果音は **初回スコープ外**
- 既存 BGM の再生状態には影響しない

## 実装境界（想定）

- 入力座標の正規化と「どのアクターが反応するか」は純粋関数として `src/features/session/` 側に置けるとテストしやすい
- スケールの時系列（elapsed → scale）も純粋関数化を推奨
- Canvas 描画ループ内で scale を乗算する。DOM の別レイヤは増やさない
- Reducer には触らない（UI エフェクトでありセッション状態ではない）

## 非スコープ（初回）

- スワイプで投げる・ドラッグで追従させる
- タップでシーン送り／色変更
- 効果音
- Setup での反応 ON/OFF
- パーティクル、星、フラッシュ演出
- 長押し・ダブルタップの別アクション

## 受け入れ条件

- `playing` 中に scene をタップすると、近い図形が短く膨らんで戻る
- `paused` / `finished` / Setup では反応しない
- 連打しても点滅や連続バウンドに見えない（レート制限が効く）
- 一時停止・終了・残り時間・シーン切替の挙動が現行と変わらない
- 色の急変や白フラッシュが起きない
