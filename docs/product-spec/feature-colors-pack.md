# 機能仕様: いろパック（colors）

いろころの初期あそび。高コントラストな背景色と図形を短時間見せ、
保護者が色や形の名前をゆっくり声に出すためのフックを提供する。

## 目的

- 「あかだね」「まるだよ」といった語りかけのきっかけを、画面側から用意する
- 赤ちゃんが視線を向けやすい高コントラストを使い、保護者の声かけと視線を重ねやすくする
- コンテンツそのものを目的化せず、あくまで人とのやり取りの補助に留める

## なぜ色と形か

| 選択 | なぜ |
|------|------|
| 原色多め・高コントラスト | 乳児期の視覚特性に寄せ、画面を「見る対象」として成立させる |
| 色 × 形の組み合わせ | 同じ色でも形が変わると語りかけのバリエーションが増える |
| 隣接シーンで色・形を連続させない | 単調さで注意が切れにくく、保護者も次の言葉を選びやすい |
| シーン dwell 6–8 秒 | 急がず、一言二言の声かけが収まる長さにする |
| 初期 pack を colors にする | いちばん説明しやすく、サービス全体のトーンを定義する入口にする |

## 語りかけの想定

保護者が画面を指さしながら、たとえば次のように使える状態を目指す。

- 「あかだね」
- 「きいろの さんかく」
- 「おほしさま みえたね」

画面は正解を求めない。名前をゆっくり言うための視覚的なきっかけでよい。

## 安全との関係

- 点滅や赤フラッシュは使わない（高コントラストでも「刺激の質」は穏やかに保つ）
- 色の急変はクロスフェードで和らげ、驚かせすぎない
- 音は別機能（BGM）であり、colors 自体は視覚と語りかけが主目的

## 受け入れの観点

- Setup から colors を選び、再生できる
- 再生中に背景色と図形が切り替わり、隣接シーンで同じ色・形が連続しない
- 保護者が色・形の名前を声に出しやすい見た目になっている

## 非スコープ

色名の自動読み上げ、クイズ形式、正解判定、保護者向けチュートリアルの強制表示。
