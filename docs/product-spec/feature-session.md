# 機能仕様: セッションライフサイクル

いろころの核は「短時間だけ開いて、保護者が主導で閉じる」セッションである。
画面やタイマーの仕組みより、なぜその区切り方が必要かを先に固定する。

## 目的

- 赤ちゃんのスクリーン時間を、保護者が意図した長さに収める
- 「つい見続けてしまう」ループを設計側で断ち、絵本や対面の関わりへ戻しやすくする
- いつでも止められる安心感を、保護者の操作負担を増やさずに提供する

## なぜこの区切りか

| 選択 | なぜ |
|------|------|
| 初期 1 分・最大 3 分 | 短い補助ツールとして使う前提。長時間視聴の代替にしない |
| 音の初期 OFF | 予期しない音で赤ちゃんや周囲を驚かせない。ON は保護者の意思が必要 |
| 一時停止で時間もシーンも止める | 「ちょっと席を外す」ときに進行が進まないことが安心につながる |
| 終了後に自動リピートしない | 見終わったあとに再び引き込まれない。再開は Setup からの意識的な操作だけ |
| Setup でのみ設定変更 | 再生中に選択肢が増えると操作が散り、見守りの邪魔になる |

## 画面の役割（何のためか）

```
Setup → Playing ↔ Paused → Finished → Setup
```

| 画面 | 役割の意図 |
|------|------------|
| Setup | あそび・時間・音を選ぶ。開始前に「どれくらい・どう遊ぶか」を決める場 |
| Playing | 語りかけの対象（色・かたち等）を短時間だけ見せる場 |
| Paused | 中断しても安全に戻れることを示す。画面も時間も止まった状態を明示する |
| Finished | 「おしまい」をはっきり区切る。続けたいときは Setup へ戻す |

## 受け入れの観点

- 開始時は常に安全側（短時間・音 OFF）から始まる
- 一時停止中は残り時間もシーンも進まない
- 終了後、操作なしに再生が再開しない
- 保護者がいつでも終了できる

## 非スコープ

自動リピート、バックグラウンド継続再生、保護者ロックなしの長時間モード。
