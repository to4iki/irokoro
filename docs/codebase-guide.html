公式設計メモ: architecture.md / product-spec / code-style.md

1. 全体像

赤ちゃん向けの短時間お遊び Web アプリ。API はなく、Cloudflare Workers Static Assets で HTML/JS/CSS だけを配信する静的アプリです。

設計の核: 画面状態は sessionReducer（純粋）に寄せる。 タイマー・BGM などの副作用は src/app.tsx に置く。

ディレクトリ責務

パス 責務
src/main.tsx React の起動入口
src/app.tsx 状態・画面切替・タイマー・BGM の司令塔
src/components/ Setup / Player / Finish / RollCanvas の UI
src/features/session/ Reducer・シーン列・転がり数学・Canvas 描画・a11y
src/content/ 色・形・どうぶつ・パック・BGM / いらすとやクレジット（データ定義）
src/audio/ HTMLMediaElement の BGM 境界
src/styles.css トークン（@theme）と CSS アニメ
src/test/ Vitest 共通セットアップ

画面遷移（状態機械）

Setup → Playing ↔ Paused → Finished → Setup
  • 設定変更（pack / duration / sound）は setup でのみ有効
  • playing は deadline と remainingMs で残り時間管理
  • paused は deadline を捨て、remainingMs だけ保持
  • finished からの再開は RESET のみ（自動リピートなし）

2. 処理の流れ

ユーザー操作 → アクション

画面 操作 dispatch 副作用（app.tsx）
Setup あそび / じかん / 音 SET_PACK / SET_DURATION / SET_SOUND なし
Setup はじめる START 音ONなら BGM 作成・再生。シーン残り時間 ref をリセット
Player 一時停止 PAUSE BGM pause。タイマー effect が止まる
Player つづける RESUME BGM play。タイマー再開
Player おしまい STOP （finished 遷移後に BGM dispose）
（自動） 残り時間 0 TICK → finished 同上
（自動） シーン dwell 経過 NEXT_SCENE なし（sceneIndex が進むだけ）
Finish はじめの画面へ RESET Setup フォーカス用 epoch を加算

タイマーは 2 系統

  1. セッション全体 — 250ms ごとに TICK。 deadline - now で remainingMs 更新。0 で finished。
  2. シーン dwell — 現シーンの durationMs（6–8 秒）後に NEXT_SCENE。pause 中は effect がアンマウントされ、残り時間を sceneRemainRef に退避。

3. モジュール詳細

入口

src/main.tsx

起動 依存: React DOM / App / styles.css

#root を取得し、createRoot(...).render(<StrictMode><App />)。 ルート要素が無いと throw。ロジックはここには置かない。

App（司令塔）

src/app.tsx — App

コンポーネント 副作用の集約
export default function App
props: 省略可能な sequence（テスト用）。 省略時は長さ 64 の createSceneSequence
内部 ref
audioRef（BGM）、 sceneRemainRef（シーン残り ms）、 setupFocusEpochRef（Finish→Setup で見出しフォーカスするか）

やっていること

  1. useReducer(sessionReducer, createInitialState) で状態を持つ
  2. scene = sequence[sceneIndex % length] で現シーンを解決
  3. status に応じて Setup / Player / Finish を描画 （Player・Finish は lazy）
  4. effects:
    • playing + deadline → 250ms TICK
    • playing → シーン setTimeout → NEXT_SCENE
    • finished → BGM dispose・sceneRemain クリア
    • playing/paused → visibilitychange で BGM pause/play
    • unmount → BGM dispose

レビュー時の要点: 副作用がここに集中しているか、Reducer に時計が漏れていないか。

src/features/session/session-reducer.ts

setup、初期値 pack=colors・60秒・音OFF

アクション早見

type 有効な status 結果
SET_PACK / SET_DURATION / SET_SOUND setup preferences 更新
START setup playing。deadline = now + duration、sceneIndex = 0
TICK playing remainingMs 更新。0 なら finished
NEXT_SCENE playing sceneIndex++
PAUSE playing paused（deadline null）。残り 0 なら finished
RESUME paused playing。deadline = now + remainingMs
STOP playing / paused finished
RESET finished 初期 setup へ

src/components/roll-canvas.tsx — RollCanvas

UI + rAF ループ

Canvas 2D の描画ループ。paused またはタブ非表示では rAF を止め、 現フレームだけ描く。リサイズは ResizeObserver。 playing 中の pointerdown（isPrimary のみ）で 「ぽん」反応を開始する（Reducer 非関与）。

props
判別 props。kind: "shape"（shapeId / shapeColor）または kind: "animal"（imageSrc）。共通で sceneId / paused
内部の流れ
createRollCast(sceneId) → 毎フレーム sampleActorPose(..., rotationStyle) （animals は tilt、colors は spin） → pose.scale に ponScaleFactor を乗算 → colors なら sampleFaceExpression も → paintRollFrame（PaintSubject）

タッチ反応「ぽん」

再生中に scene（Canvas）を触ると、近い図形が短く膨らんで戻る。 セッション進行・deadline・シーン切替には影響しない UI エフェクト。 仕様の「なぜ」は feature-touch-pon.md。

4. データの流れ図

main.tsx
  └─ App
       ├─ sessionReducer  ←── dispatch(アクション)
       │     └─ SessionState (setup|playing|paused|finished)
       │
       ├─ DEFAULT_SEQUENCES[packId][sceneIndex] → Scene
       │     (colors: shapeId / animals: animalId)
       │
       ├─ SetupScreen ← preferences / onChange* / onStart
       ├─ PlayerScreen ← state + scene
       │     └─ SceneStage (memo)
       │           └─ RollCanvas (kind: shape | animal)
       │                 ├─ pointerdown → touch-pon.ts（対象選定・scale）
       │                 ├─ createRollCast / sampleActorPose(…, spin|tilt)
       │                 ├─ sampleFaceExpression（colors のみ）
       │                 └─ paintRollFrame(PaintSubject)
       │                       └─ 色+形+かお or getAnimalImage
       └─ FinishScreen ← onReset

副作用（App の useEffect / ハンドラ）:
  Date.now タイマー → TICK / NEXT_SCENE
  createBackgroundMusic → play / pause / dispose
  document.visibilitychange → BGM

5. レビューの観点（次に見るとき）

  1. 境界 — ルールは Reducer、副作用は App、見た目は components、 データは content に収まっているか
  2. 安全制約 — 初期 1 分・音 OFF・自動リピートなし・ finished 後は RESET のみ（product-spec）
  3. タイマー二系統 — pause / タブ非表示でセッション進行・シーン・ Canvas・BGM が止まるか
  4. 純粋さ — reducer / sequence / roll に DOM や Date.now() が混ざっていないか（now はアクション引数で渡す）
  5. テスト先行 — 遷移変更なら失敗テストが先か
  6. パック分岐 — Scene / RollCanvas / paint の判別が colors と animals で一貫しているか
  7. ぽん — 触覚反応がセッション時間やシーン進行を 変えていないか
  8. かお — ゆるいモーフと安全制約（点滅なし）が 守られているか。animals に載せていないか

6. 用語メモ

言葉 意味

パック あそびの種類。colors と animals
tilt どうぶつ用の弱い回転（±12°）。spin の上下逆を避ける

FSM 有限状態機械。ここでは session の status 遷移
ぽん playing 中の短いスケール反応。Reducer 外の Canvas エフェクト
かおモーフ smile → surprise のパラメータ補間。colors 図形専用
